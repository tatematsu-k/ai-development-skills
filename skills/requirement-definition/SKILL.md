# 要件定義 (Requirement Definition)

## Overview

ユーザーの「やりたいこと」を対話を通じて明確な要件に落とし込む。

**Announce at start:** 「要件定義フェーズを開始します。やりたいことを詳しく教えてください。」

## The Process

### Step 1: コンテキスト把握

1. 既存のコードベース・ドキュメントを確認する
2. ブラウンフィールド(既存プロジェクトへの追加)か、グリーンフィールド(新規)かを判定
3. ブラウンフィールドの場合、関連するコードの構造を把握する

### Step 2: 要件の深掘り

ユーザーと1問ずつ対話して要件を明確にする。以下の観点を網羅する:

- **What**: 何を実現したいか
- **Why**: なぜそれが必要か (ビジネス上の理由)
- **Who**: 誰が使うか (ユーザー・システム)
- **Scope**: スコープ (何を含み、何を含まないか)
- **Constraints**: 制約条件 (技術的・ビジネス的)
- **Success Criteria**: 完了条件

**質問のルール:**
- 1メッセージにつき1問
- 可能な限り選択式で提示
- 推測できるものは提案して確認を取る

### Step 3: 要件定義書の作成

以下のテンプレートで作成する:

```markdown
# [機能名] 要件定義書

## 1. 概要
- 何を実現するか (1-2文)

## 2. 背景・目的
- なぜ必要か
- ビジネス上の期待効果

## 3. スコープ
### 含むもの
- ...

### 含まないもの
- ...

## 4. 機能要件
### FR-1: [要件名]
- 説明: ...
- 受け入れ条件: ...

## 5. 非機能要件
- パフォーマンス: ...
- セキュリティ: ...
- その他: ...

## 6. 制約条件
- 技術的制約: ...
- ビジネス制約: ...

## 7. 用語定義
| 用語 | 定義 |
|------|------|
| ... | ... |
```

### Step 4: 保存とコミット

1. `aidlc-docs/requirements/YYYY-MM-DD-<feature>.md` に保存
2. gitにコミット
